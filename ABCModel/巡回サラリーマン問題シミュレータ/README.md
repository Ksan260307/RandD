# 巡回サラリーマン問題シミュレータ

営業マンが複数の顧客を訪問する際の「最適ルート」を、従来の「物理的最短距離」ではなく、**ABCモデルに基づくメンタルヘルスの視点**から再考するシミュレーション。心理的破綻（RuinScore）を最小化する真の最適解を発見します。

## 概要

古典的な「巡回セールスマン問題（Traveling Salesman Problem: TSP）」を拡張し、営業マンの内部心理状態（A: 生感、B: 評価圧、C: 監視認知）がいかにルート選択に影響するかを可視化します。

**効率だけを追求する最短ルート** vs **メンタル保護を優先する最適ルート**を同時にシミュレートし、どちらが本当に最適かを判断できます。

## 主要機能

### ABCモデル統合エンジン

各ノード訪問時に、以下のABCパラメータが動的に変化します：

#### A: Authenticity（生感・行動力）
- **変化**: ストレス（E0）で低下、良い環境（E1）で回復
- **自然減衰**: 目標値（50）に徐々に収束
- **範囲**: 0-100

#### B: Aesthetic（評価圧・プレッシャー）
- **変化**: 社会圧（E2）で上昇
- **自然減衰**: 目標値（0）に徐々に収束
- **危险トリガー**: B > 80 かつ C > 80 で+30追加ダメージ
- **範囲**: 0-100

#### C: Meta（監視認知・自己意識）
- **変化**: タスク過多（E3）と体調悪化（E0）で上昇
- **自然減衰**: 目標値（0）に徐々に収束
- **範囲**: 0-100

### 環境ストレス要因（EF）

各ノードに設定された外部要因により、ABCパラメータが影響を受けます：

| ノード | 説明 | E0 | E1 | E2 | E3 | 備考 |
|--------|------|-----|-----|-----|-----|------|
| **自社** | 拠点 | 0 | 0 | 0 | 0 | 中立 |
| **顧客A** | 通常営業先 | 1 | 0 | 2 | 1 | 低ストレス |
| **顧客B** | 激怒した顧客 | 2 | 0 | 6 | 5 | **極度のストレス** |
| **顧客C** | VIP重鎮顧客 | 1 | 0 | 4 | 2 | 中程度ストレス |
| **カフェ** | 純喫茶（休息） | -2 | 8 | -2 | -2 | 回復スポット |
| **公園** | 自然公園（休息） | -1 | 5 | -3 | -5 | 強力な回復スポット |

**凡例**:
- E0: 体調・睡眠の負荷
- E1: 心地よい環境
- E2: 社会圧・評価圧
- E3: タスク過多・負荷

### ルート選択

#### ルート1: 物理的最短ルート（効率重視）
```
自社 → 顧客A → 顧客C → 顧客B → 自社
```

**特徴**:
- 移動距離が短い
- **危険性**: 連続的なストレス（特に顧客B）により、B（評価圧）とC（監視）が暴走
- **結果**: メンタル崩壊の危機状態

**推奨不可**: このルートはRuinScoreが著しく高くなる可能性があり、メンタルヘルス上の最悪ケース

#### ルート2: ABC最適化ルート（メンタル保護）
```
自社 → 顧客A → カフェ(休息) → 顧客C → 公園(休息) → 顧客B → 自社
```

**特徴**:
- 移動距離は長い
- **保護機構**: ストレスの間にカフェや公園を挟むことで、E1（良い環境）の効果により回復
- **メカニズム**: 
  - A（生感）を維持することで、B（評価圧）への制御効果が発動（A→B相互作用）
  - 連続ストレスによる暴走を予防
- **結果**: RuinScoreが大幅に低下し、メンタル安定状態を保持

**推奨**: メンタルヘルスを優先する企業・管理者の選択肢

### RuinScore（心理的破綻リスク）

心理状態の総合危険度（0-100）：

```
基本 RuinScore = max(0, (100-A)×0.5, B×0.8, C)
特殊条件: B > 80 かつ C > 80 なら +30 ボーナスダメージ
```

| スコア範囲 | 状態 | 説明 |
|-----------|------|------|
| 0-30 | 安定 | メンタル良好、業務継続可能 |
| 30-50 | 注意 | ストレス蓄積、休息推奨 |
| 50-70 | 危険 | 判断力低下、ミス増加の懸念 |
| 70+ | 破綻 | 即座の休息・外部支援が必要 |

### シミュレーション状態

#### 時系列シミュレーション

1. **過去区間**: これまで訪問したノードの累積影響を反映
2. **現在ノード**: 立ち寄り中のノードの環境要因を適用
3. **移動負荷**: 区間移動中の連続ストレス（E0）を段階的に加算
4. **リアルタイム計算**: 進捗率に応じた正確な状態変化

### UI表示

#### グラフィカル可視化

- **マップ**: ノード位置をプロット、現在地を動きながら表示
- **ABCメーター**: リアルタイム変化を3つの独立したバーグラフで表示
- **RuinScore**: 危険度を単一の大型メーター＋危険アラートで表示
- **進捗バー**: ルート進行状況を0-100%で表示

#### 操作パネル

- **ルート選択**: 2つのルート間を切り替え可能
- **再生/一時停止**: `▶ 再生`/`⏸ 一時停止`ボタン
- **リセット**: `🔄 再度実行`で初期状態に戻す

## 使用方法

### 基本操作

1. **ルート選択**: 
   - 「物理的最短ルート」または「ABC最適化ルート」を選択
   - 説明文とノード配置が切り替わる

2. **シミュレーション実行**:
   - `▶ 再生`ボタンをクリック
   - 営業マンがマップ上を移動
   - 各ノードで停止し、ストレスを受ける

3. **状態監視**:
   - **A（生感）**: 高いほど良好（目標: 50以上）
   - **B（評価圧）**: 低いほど良好（目標: 0に近い）
   - **C（監視）**: 低いほど良好（目標: 0に近い）
   - **RuinScore**: 低いほど安全

4. **比較分析**:
   - 2つのルートを交互に実行
   - 最終的なRuinScoreの差を観察
   - メンタル保護ルートの効果を検証

### 関連計算式

#### ABCモデル更新

```javascript
nextA = A - (E0 × 0.5) + (E1 × 2)
nextB = B + (E2 × 2.5)
nextC = C + (E3 × 1.5) + (E0 × 0.2)
```

#### 相互作用（相殺効果）

```
C > 60 → A -= (C - 60) × 0.3    // 過度な監視がエネルギー喪失
B > 50 → C += (B - 50) × 0.2    // プレッシャーが自己監視強化
A > 60 → B -= (A - 60) × 0.4    // 生感がプレッシャー軽減
```

#### 自然減衰（ホメオスタシス）

```
nextA = nextA + (50 - nextA) × 0.05
nextB = nextB + (0 - nextB) × 0.05
nextC = nextC + (0 - nextC) × 0.1
```

## 実学的な応用

### 企業向け

- **営業効率化**: 従来の「距離最適」から「人間中心」へのシフト
- **メンタルケア**: 営業スタッフのための科学的な巡回ルート設計
- **生産性維持**: 長期的なパフォーマンス向上（短期効率より持続性）

### 教育・研究

- **心理学**: 環境ストレスと心理状態の可視化
- **経営学**: 人的資源管理としてのルート設計
- **組織行動**: 意思決定における複合因子の検討

## 注記

- このシミュレーションは教育目的のモデルです
- 実際の営業ルート設計には、さらに多くの変数の検討が必要です
- RuinScore の絶対値よりも、**2つのルートの相対的な差**を比較観察することが重要です
- メンタルヘルスは多因子的なため、このツールだけで判断すべきではありません
