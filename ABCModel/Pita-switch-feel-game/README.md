# Pita-switch-feel-game（ピタスイッチ フィールゲーム）

心理パラメータを調整して、ボールが理想的に流れるコース配置を作るゲーム。**流動性と秩序のバランス** を、ビジュアルとスコアを通じて実感的に学べるインタラクティブ体験です。

## 概要

スマートボール／ピンボール的な仕組みを使いながら、複雑な心理力学（ABCモデル）を **ゲームプレイの快感** として表現します。

プレイヤーは：
1. **6つのモジュール** を6つのスロットに配置
2. **6つのパラメータスライダー** で環境を調整
3. ボールが「するする」と心地よく流れるバランスを探索
4. スコアと「心地よさ度（pleasant）」で結果を評価

という体験を通じて、個人の心理パラメータと、相互作用がいかに複雑に絡み合うかを **直感的に** 理解できます。

## 主要機能

### 7種類のモジュール

各モジュールはボールの流れに異なる影響を与えます：

| モジュール | 絵文字 | 効果 | 説明 |
|-----------|--------|------|------|
| **さかみち（Ramp）** | 🛝 | 大きなゲイン | ボールを加速させる坂。最もスコアが出やすい |
| **ドミノ** | 🁢 | 連鎖ボーナス | flow が十分あれば連鎖（combo +2）。失敗のリスク |
| **はずみ玉（Bumper）** | 💥 | バランス依存 | order のバランスが良いとボーナス |
| **ふうせん風（Fan）** | 💨 | breeze 依存 | そよ風パラメータが高いほど効果的 |
| **くるり板（Spinner）** | 🌀 | order+guide依存 | 指針がそろってると高得点。揃わないと低め |
| **ベル（Bell）** | 🔔 | 爽快感 | smooth が高いほど気持ちいい。最高得点 |
| **なにもなし（Blank）** | ▫️ | 休息 | 息つく間。負の影響なし |

### 6つの心理パラメータ

#### 1. ころがる勢い（push）
- **意味**: ボールの初速、人生のエネルギー
- **範囲**: 0-100
- **効果**: flow を直接増加、でも高すぎるとコントロール困難

#### 2. きっちり度（neat）
- **意味**: 秩序立て、完璧さへのこだわり
- **範囲**: 0-100
- **効果**: order を増加。高すぎると「考えすぎ」に

#### 3. 見守りアシスト（guide）
- **意味**: 外部からの優しい導き、サポート
- **範囲**: 0-100
- **効果**: guide を増加。高すぎると「過保護」に

#### 4. そよ風（breeze）
- **意味**: 環境的な穏やかさ、微妙な助け
- **範囲**: 0-100
- **効果**: flow を低下させる（緩やかに）、fan の効果を増加

#### 5. プレッシャー（pressure）
- **意味**: 外部からのストレス、期待
- **範囲**: 0-100
- **効果**: order と guide を上昇させる（焦りで慎重に）

#### 6. ざわつき（noise）
- **意味**: 雑音、注意散漫、混乱
- **範囲**: 0-100
- **効果**: breeze 同様に flow を低下、wobble を上昇

### 内部計算パラメータ

ユーザーのスライダーから計算される **見えない層**：

#### flow（流動性）
```
flow = clamp(push - 0.28 × (breeze + noise), 0, 2)
```
ボールがスムーズに流れるエネルギー。高いほど心地よい。

#### order（秩序）
```
order = clamp(neat + max(0.32 × pressure, 0.18 × noise), 0, 2)
```
システムの整然さ。適度だと良好だが、high かつ flow 低いと「考えすぎ」。

#### guide（導き）
```
guide = clamp(guideBase + ..., 0, 2)
```
外部サポートの度合い。高いと smooth を上昇、流れをサポート。

#### wobble（揺らぎ）
```
wobble = clamp(max(breeze, noise) + ..., 0, 2)
```
不安定性。高いと jam を加速し、smooth を低下。

#### jam（詰まり）
```
jam = clamp(ペナルティ条件の and/or, 0, 2)
```
システムが「つっかえている」度合い。

#### smooth（滑らかさ）
```
smooth = clamp(
  flow × 0.75 + (2 - |order - 1|) × 0.42 + (2 - |guide - 1|) × 0.28 - jam × 0.55,
  0,
  2
)
```
全体的な心地よさの「滑らかさ」。これが高いほど良い。

#### pleasant（心地よさ度）
```
pleasant = clamp(round(54 + smooth × 18 - jam × 15), 0, 100)
```
0-100の数値で「現在のバランスがどれだけ心地よいか」を表現。

### スコアシステム

各モジュールごとに異なるスコア：

| モジュール | 基本スコア | ボーナス条件 |
|-----------|----------|----------|
| さかみち | 72 | flow による加算 |
| ドミノ | 84 | 連鎖成功時 |
| はずみ玉 | 68 | order バランス |
| ふうせん風 | 62 | breeze パラメータ |
| くるり板 | 76 | order+guide 依存 |
| ベル | 88 | smooth が高い時 |
| なにもなし | 16 | 最小スコア |

**ゴールボーナス**:
```
finalBonus = 120 + bestCombo × 18 + smooth × 30
pleasantBonus = (vx / 5) + pleasant × 0.3
```

### リアルターム評価メッセージ

スライダー調整に応じて、AIが状態を分析して "mood" と "tip" を提示：

| 状態 | Mood | Tip |
|------|------|-----|
| 高 smooth | 「かなりするする」 | 「ベルやドミノを増やすとさらに爽快」 |
| jam > 1.45 | 「考えすぎて少しつっかえやすい」 | 「見守りアシストか きっちり度を下げる」 |
| flow < 0.55 | 「勢いが足りず止まりやすい」 | 「ころがる勢いを上げるか ざわつきを下げる」 |
| order 高 + pressure 高 | 「きっちりしすぎで慎重モード」 | 「プレッシャーを少し下げる」 |

## 使用方法

### ゲームプレイの流れ

1. **初期状態**
   - デフォルトの6つのモジュール配置が表示
   - 6つのスライダーが中程度の値に設定

2. **モジュール配置の変更**
   - 左下の「モジュール選択」パネルでモジュールをクリック
   - スロット（上の6つの箱）をクリックして、そのスロットにモジュールを配置
   - 複数のスロットに同じモジュールも配置可能

3. **パラメータ調整**
   - 右側の6つのスライダーをドラッグして調整
   - リアルタイムで「心地よさ度」と推奨メッセージが更新
   - スライダーを変更すると、ボールの初期位置が計算し直される

4. **ゲーム開始**
   - 左上の「スタート」ボタンをクリック
   - ボールが左から右に流れる
   - 各モジュールを通過するたびにパーティクル（✨、🟨、💥など）が発生

5. **結果確認**
   - ボールがゴール（右端）に到達したら finish
   - または、2.3秒以上速度が低下（stall > 2.3）したら failure
   - スコア、デアコンボ、「心地よさ度」が表示

6. **再ゲーム**
   - モジュール配置やパラメータを調整
   - 「スタート」ボタンで再挑戦

### インタラクティブな実験

#### 実験1: 「するする」を目指す
```
スライダー設定:
- ころがる勢い: 70
- きっちり度: 50
- 見守りアシスト: 60
- そよ風: 30
- プレッシャー: 20
- ざわつき: 10

期待結果: pleasant ≈ 75-80（"かなりいい感じ"）
モジュール配置: ベル、ドミノ、くるり板を増やす
```

#### 実験2: 「考えすぎ」を体験
```
スライダー設定:
- ころがる勢い: 50
- きっちり度: 90（高い！）
- 見守りアシスト: 80
- そよ風: 20
- プレッシャー: 70
- ざわつき: 15

期待結果: pleasant ≈ 40-55（"ちょい調整で化ける"）
現象: ボールが途中で止まりやすい、stall が発生
```

#### 実験3: 「そよ風」の効果
```
比較:
- ざわつき: 10 & そよ風: 10 → pleasant ≈ 70
- ざわつき: 50 & そよ風: 10 → pleasant ≈ 35
- ざわつき: 50 & そよ風: 80 → pleasant ≈ 55

洞察: そよ風（穏やか）はざわつき（雑音）を相殺できる
```

#### 実験4: 「プレッシャー」の影響
```
シナリオ: 外部からのストレスが増加した場合
- プレッシャー: 0 → pleasant ≈ 75
- プレッシャー: 50 → pleasant ≈ 65（order 増加で慎重に）
- プレッシャー: 100 → pleasant ≈ 45（過度な緊張）

学び: プレッシャーは order を上げるが、flow 低下につながる
```

## 心理学的背景

このゲームは、複数の心理パラメータが相互に干渉し合う **非線形システム** をシミュレートしています：

### 相互作用の例

1. **C→A（監視が行動力を奪う）**
   ```
   guide が高い & flow が低い → stall（詰まり）が発生
   解決策: push を上げるか、guide を下げる
   ```

2. **B（秩序）が高すぎる**
   ```
   order > 1.45 でも guide を追加で上昇
   結果: ボールが「考えすぎて」進まない
   ```

3. **流動・秩序・導きのバランス**
   ```
   smooth = flow × 0.75 + order距離 + guide距離 - jam
   最適: flow ≈ 1-1.5, order ≈ 1, guide ≈ 1
   ```

## 応用例

- **自己認識**: 自分の心理パラメータを設定してシミュレーション
- **人間関係**: 相手のパラメータを予測して、どう対応するかを検討
- **組織運営**: チーム全体の flow / order / wobble を最適化
- **教育**: 複雑なシステムの相互作用を直感的に学ぶ
- **ゲーム化**: つまらない心理学を楽しい体験に変換

## 注記

- このゲームは、パラメータの**相対的な影響**を直感的に理解するためのものです
- pleasant スコアや jam の計算は、実在の心理測定ツールではなく、**ゲームデザイン上の値**です
- 個々の人間の複雑さを完全には捉えられません
- 現在は **6つのスロット固定**ですが、スロット数や種類を拡張可能な設計になっています
