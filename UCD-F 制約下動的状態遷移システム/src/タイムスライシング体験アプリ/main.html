タイムスライシング（計算の分割処理）体験

「未来の変動予測」のような膨大で重いデータ解析を一度に行うと、画面が一時的に「フリーズ」してしまいます。
計算タスクを細かく分け、描画の合間に少しずつ進めることで、画面を滑らかに保つ技術を体験してみましょう。

画面の滑らかさ確認アニメーション

重い計算中に上のブロックやアイコンの動きが止まったら、画面（メインスレッド）がフリーズしている証拠です。

一括処理 （従来の方法）

1500個のデータブロック解析を一度に行います。
完了するまで画面が操作不能（フリーズ）になります。

待機中

0.00s

裏側で実行中の解析タスク (1500個) 0 / 1500

※一括処理では計算中、画面の更新処理も停止するため、ブロックが塗られていく途中経過は見えません。 

// すべての計算を1つのループで一気に実行
for (let i = 0; i < TOTAL_STEPS; i++) {
    doHeavyWork(); // 重い処理
}

// ↑ このループが終わるまで画面（メインスレッド）は占有され、
// 進捗バーの更新やアニメーションは一切行われません。
updateUI(); // 終わってから初めて画面が更新される

分割処理 （タイムスライシング）

解析を細かく区切り、描画の合間に少しずつ実行します。
アニメーションは止まらず、進捗も目視できます。

待機中

0.00s

裏側で実行中の解析タスク (1500個) 0 / 1500

黄色く光っている箇所が、まさに今ブラウザの裏側で計算・解析処理を行っているデータブロックです。 

function processChunk() {
    const start = performance.now();
    
    // 1回の描画の合間（約12ミリ秒）だけ計算する
    while (step < TOTAL && (performance.now() - start) < 12) {
        doHeavyWork();
        step++;
    }
    
    updateUI(); // UI（進捗バーなど）をここで更新

    if (step < TOTAL) {
        // 残りの計算は、次の描画タイミングに回す（画面は滑らか）
        requestAnimationFrame(processChunk);
    }
}
requestAnimationFrame(processChunk); // 処理開始
