🛡️ 防衛システムの仕組み (AAIMA vs 従来型FW)

1. 未知の脅威「同化プローブ」とは？

本シミュレーションで発生する赤い攻撃は、単にデータを壊すだけのウイルスではありません。デバイスの内部に入り込み、完全に「自分たちの仲間（攻撃者）に乗っ取ってしまう」非常に高度な攻撃です。
乗っ取られたデバイス（赤色）は、今度は自分自身が周囲の正常なデバイスに向かって次々と攻撃を放ち始めます（感染爆発）。

2. 従来型ファイアウォール（FW）の弱点

過去のウイルスリスト（ブラックリスト）と照らし合わせて防ぐ、従来からある防衛方法です。

  • リストにない未知の攻撃（同化プローブ）はほとんど防ぐことができず、素通りさせてしまいます。
  • 最大の弱点： 自分のデバイスが乗っ取られたことに気づけません。そのため、乗っ取られたデバイスが新たな攻撃の拠点となってしまい、あっという間にネットワーク全体が赤く染まって崩壊してしまいます。

3. 次世代型「AAIMA（アイマ）」の強み

最新の「ゼロ知識証明（ZKP）」という技術を使い、内部の情報を一切渡さずに「自分は正常だ」と証明することで攻撃を弾き返します（緑の波紋エフェクト）。しかし、AAIMAの本当の強さは別にあります。

  • 【最大の強み：自律隔離（トカゲの尻尾切り）】 万が一、防御を突破されてデバイスが乗っ取られそうになっても、AAIMAはデバイス自身に危険を察知させます。完全に手遅れになって周囲を攻撃し始める前に、自らネットワークの通信コードを引き抜き、データを消去して沈黙します（黄色く光り、線が消える）。
  • 【結果】 一つのデバイスが犠牲になることは防げませんが、それが新たな攻撃拠点になることを防ぐため、被害の連鎖（感染爆発）を確実に食い止め、ネットワーク全体を守り抜くことができます。
