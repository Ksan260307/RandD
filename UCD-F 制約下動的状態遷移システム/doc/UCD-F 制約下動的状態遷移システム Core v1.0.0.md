# 制約下動的状態遷移システム（UCD-F）設計書

## 1. 概要

本設計書は、極端な制約環境下におけるシステム設計思想を基盤とし、状態遷移・可視性制御・力学（Dynamics）を統合した汎用フレームワーク「Universal Constrained Dynamics Framework（UCD-F）」の仕様を定義する。

本フレームワークは以下の特徴を持つ：

* 状態を固定長構造として扱う
* 制約を前提とした設計（メモリ・権限・計算資源）
* 量子化を用いたデータ表現と通信・演算の極限最適化
* 可視状態（Bank）によるアクセス制御
* 状態変化に「速度・強度・疲労（VIF）」の三要素を付与
* 実行環境に応じた自律的なパフォーマンス最適化
* 予測並行プロセス（Shadow Worker）による未来探索
* 階層構造（Hierarchy）を維持した高速なSoA演算
* ネットワーク合意モデルによる決定論的ロールバック
* スキーマコンパイラによるメモリレイアウトの自動生成
* イベント駆動と純粋関数による完全な状態の可逆性とスケーラビリティ

---

## 2. システム全体構造

システムは以下の12要素を核として動作する。パフォーマンスと安定性を担保するため、コアロジック、描画制御、未来予測、I/O処理はそれぞれ独立したプロセス（Web Worker等）で並行実行される。

System = $(S, A, C, B, D, M, Q, \epsilon, V, P, H, L)$

| 要素 | 内容 |
| --- | --- |
| $S$ | 状態集合（State） |
| $A$ | アクション（Action） |
| $C$ | 制約（Constraint） |
| $B$ | 可視性制御（Bank） |
| $D$ | 力学（Dynamics） |
| $M$ | メトリクス（Metrics） |
| $Q$ | 量子化（Quantization） |
| $\epsilon$ | 環境ノイズ（Environmental Noise） |
| $V$ | 描画アダプタ（Viewport Adapter） |
| $P$ | 予測エンジン（Shadow Worker） |
| $H$ | 階層トポロジー（Hierarchy Topology） |
| $L$ | メモリレイアウトスキーマ（Layout Schema） |

---

## 3. 状態（State Layer）

### 3.1 定義とSoA構造

各状態は固定長ベクトルとして定義する。計算効率とキャッシュヒット率を極限化するため、状態の実体は **SharedArrayBuffer** 上に確保された **TypedArray** を用いた **SoA（Structure of Arrays）** 形式で管理する。
すべての内部状態は、独自の **スキーマコンパイラ（DSL）** を通じて生成されたオフロード管理クラスによって、ビットレベルでの厳密なレイアウトが保証される。

### 3.2 階層構造と更新順序（Topological Sort）

フラットなSoA構造において親子関係（連結体）を維持するため、システムは **トポロジカル・ソート済みの更新インデックス配列** を保持する。演算ループはこの配列を参照して実行され、親エンティティの状態確定後に子エンティティを演算することで、SoAの速度を維持しつつ階層間の伝播遅延（1フレーム遅れ）を完全に排除する。

### 3.3 空間インデックス（Spatial Hashing）

大量のエンティティから可視範囲や相互作用対象を高速に抽出するため、SoA構造と同期した空間ハッシュを保持し、計算量を平均 $O(1)$ または $O(\log n)$ へ削減する。

### 3.4 階層型メモリ管理（Hot / Warm / Cold）

1. **Hot**: `SharedArrayBuffer` 上のSoA。全スレッドからゼロコピー参照される。
2. **Warm**: メインメモリ上の待機状態。
3. **Cold**: I/O専用Workerを介してIndexedDBへオフロードされた状態。チャンク分割によりループをブロックしない。

---

## 4. 量子化（Quantization Layer）

### 4.1 動的精度調整（LOD）

量子化 $Q$ は単なる圧縮ではなく、負荷 $M(t)$ に応じた動的精度調整エンジンである。注視点からの距離やシステムの負荷状況に応じ、演算および通信の精度を動的に落とすことで、計算リソースを最適化する。

---

## 5. アクション（Action Layer）

### 5.1 アクション・リングバッファ

入力によるGC（ガベージコレクション）スパイクを回避するため、アクションの送受信には **SharedArrayBuffer上の固定長リングバッファ** を使用する。入力をゼロアロケーションで処理し、スレッド間の通信オーバーヘッドを最小化する。

### 5.2 ネットワーク合意モデル

マルチプレイ環境において、システムは **決定論的ロールバック（GGPO方式）** または **権威サーバーモデル** のいずれかを選択する。全クライアントが同一の Tick ID で確定的演算を実行し、不整合検知時には過去の正常スナップショットへ巻き戻して再演算（Reconciliation）を行う。

---

## 6. 力学モデル（Dynamics Layer）

### 6.1 VIF（Velocity, Intensity, Fatigue）モデル

状態遷移には慣性に加え、一時的な熱量を示す **強度（Intensity）** と、長期的・不可逆的な蓄積を示す **疲労（Fatigue）** を組み込む。これら三要素はすべて量子化され、スナップショット保護の対象となる。これにより、複雑な非線形挙動を再現可能かつ確定的なものとする。

---

## 7. 描画・UIアダプタ（View Layer）

### 7.1 ダイレクト・バインディング

描画アダプタ $V$ は、SoAデータをWebGL/WebGPUの **InstancedMesh** へ直接バインドする。JSオブジェクト空間を経由しないため、数万規模のエンティティ描画でもゼロアロケーションを維持する。

### 7.2 UI/UXの分離

内部の複雑な物理パラメータは、アダプタ層で「平易な言葉（Plain Language）」と「直感的なスライダー」にマッピングされる。ユーザーが操作する正規化された値（0.0〜1.0）は、内部でUCD-Fの固定小数点パラメータへと自動変換される。

---

## 8. 予測・探索（Shadow Layer）

### 8.1 Shadow Worker

メインエンジンとは別に、現在のスナップショットを複製して未来を高速試行する **Shadow Worker** を稼働させる。AIの行動探索やプレイヤーの軌跡予測を、本番のタイムラインを汚染せずに並行処理する。

---

## 9. システムフロー

1. **スキーマ生成**: DSLからメモリレイアウトとオフセット管理クラスを自動生成。
2. **メトリクス収集**: 負荷 $M(t)$ に基づき、量子化解像度を決定。
3. **アクション登録**: リングバッファへ入力をゼロアロケーションで書き込み。
4. **階層演算**: トポロジカル・ソート済み配列に基づき、親子関係を維持した確定的演算を実行。
5. **未来探索**: Shadow Workerによる並行予測シミュレーション。
6. **状態確定**: Tick IDに基づき、ネットワーク合意を介して不整合を補正。
7. **描画更新**: InstancedMeshへのダイレクト・バインディング。
8. **非同期退避**: I/O WorkerによるCold Storageへのバックグラウンド書き込み。

---

## 10. まとめ

本設計は、SoA構造、階層型更新、ゼロアロケーション描画、そして未来予測を統合することで、計算資源の制約を技術的優位性に転換するものである。モバイル環境等の厳しい条件下においても、高度な物理シミュレーションと安定稼働を両立させる。
