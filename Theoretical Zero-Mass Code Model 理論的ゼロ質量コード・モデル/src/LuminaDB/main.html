          API Playground & Integration
        
    
        
      
    
      
      
    
        
        
    
          
    1. Internal Browser Interception
    
          
    
            左ペインのAPIをONにすると、ブラウザコンソールからの fetch('/api/v1/query') や、他ウィンドウからの postMessage をLuminaDBが傍受して応答します。
          
    
          
    
            POST
            /api/v1/query
          
    
          
          
    
            
          
    
          
    Awaiting request...
    
        
    

        
        
    
          
    
            
            2. External Access (Node.js Standalone Server)
          
    
          
    
            ブラウザのセキュリティにより、HTMLから直接TCPポートを開いて外部からのリクエストを受信することはできません。
    
            外部のプログラム（Pythonスクリプトやcurlコマンド等）からLuminaDBを直接叩きたい場合は、現在のデータとコアエンジンを内包したスタンドアロンのNode.jsサーバーをエクスポートしてください。
          
    
          
    
            $ node luminadb_server.js
            $ curl -X POST http://localhost:8080/api/v1/query -H "Content-Type: application/json" -d '{"query":"SELECT * FROM users"}'
          
    
          
    
            
          
    
        
    
      
    
    
    
  
    

  
  
  
  
    
    
    
    
    
      
    
        
    
          
          Data Generator
        
    
        
      
    
      
      
    選択したテーブルにダミーデータを10,000件注入します。実行後、システムのデータ断片化が増加します。
    
      
      
    
        
        
      
    
      
      
    
        
      
    
    
    
  
    

  
  
  
  
    
    
    
    
    
      
    
        
    
          
          CSV Import
        
    
        
      
    
      
      
    
        ローカルのCSVデータを既存のテーブルに取り込み（追加）します。
    
        ※対象テーブルのカラム数とCSVの列数が完全に一致している必要があります。（1行目はヘッダーとして無視されます）
      
    
      
      
    
        
        
      
    

      
    
        
        
      
    
      
      
    
        
      
    
    
    
  
    

  
  
  
  
    
    
    
    
    
      
    
        
    Engine Settings
    
        
      
    
      
    
        
    
          
    
            
            In-Process
          
    
          
          
    0に近づくほどネットワーク層をバイパスし応答速度が最大化します。
    
        
    
        
    
          
    
            
            100%
          
    
          
          
    メモリ上のデータ圧縮率です。高密度に配置するほどキャッシュ効率が上がります。
    
        
    
        
    
          
    
            
            Strict (Lock-free)
          
    
          
          
    並行処理時の保護レベルです。 Strictにすると安全に高速処理されます。
