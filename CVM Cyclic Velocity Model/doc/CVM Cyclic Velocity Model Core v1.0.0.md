# Cyclic Velocity Model（CVM）設計書

## 1. 概要

本設計書は「生命の循環構造」を抽象化し、あらゆる対象（個人・組織・プロダクト・コンテンツ）に適用可能な汎用フレームワークとして定義するものである。

本モデルは以下の特徴を持つ：

* 状態遷移によるライフサイクル管理
* 10段階のベロシティ（活動強度）による動的評価
* 累積量（生命総量）の最大化による終了条件
* サイクルを跨いだ進化（アドバンテージ継承）

---

## 2. コアコンセプト

### 2.1 循環構造

対象は以下の5状態を循環する：

* Birth（誕生）
* Growth（成長）
* Maturity（成熟）
* Decline（衰退）
* Death（死）

この遷移は線形ではなくループ構造を持つ。

---

### 2.2 ベロシティ（Velocity）

ベロシティは対象の「活動強度」または「エネルギー流量」を表す離散スケール：

* 定義域：1〜10（整数）

| レベル | 定義          |
| --- | ----------- |
| 1–2 | 停滞・潜伏       |
| 3–4 | 低活動         |
| 5–6 | 安定活動        |
| 7–8 | 高活動         |
| 9   | 飽和直前        |
| 10  | 最大状態（死フェーズ） |

重要な制約：

* **Deathフェーズにおいてベロシティは必ず最大値（10）となる**

---

### 2.3 生命総量（Life Accumulation）

対象の内部状態として累積値を定義する：

* 記号：L
* 意味：経験・情報・エネルギーの総量

更新ルール：

L(t+1) = L(t) + f(V(t))

fは単調増加関数（例：線形、指数など）

---

### 2.4 死の定義

死は減衰ではなく「最大化」である：

* 条件：
  L ≥ L_max

* 状態：
  Death時にベロシティは10

---

### 2.5 再誕と進化

新しいサイクルは前回の結果を引き継ぐ：

V_init(next) = V_init(current) + α

または一般化：

θ(next) = θ(current) + g(L_max)

これによりシステムは以下の性質を持つ：

* 非定常性（毎サイクル変化）
* 学習能力
* 進化的最適化

---

## 3. 数理モデル

### 3.1 状態

S(t) ∈ {B, G, M, D, X}

---

### 3.2 ベロシティ更新

V(t+1) = V(t) + Δ(S(t), E(t)) + ε

* Δ：状態依存の変化量
* E：外部環境
* ε：ノイズ

---

### 3.3 累積

L(t+1) = L(t) + f(V(t))

---

### 3.4 終了条件

if L(t) ≥ L_max:
S(t) = Death

---

### 3.5 再初期化

S(0) = Birth
V(0) = V_init + α

---

### 4.2 Velocity Engine

---

* 過去サイクルの記録
* パラメータ更新
* 学習ロジック

## 10. 本質

本モデルは以下を前提とする：

「終わりは消滅ではなく、最大化である」

この思想により、

* 死＝ピーク
* 再誕＝進化

という構造を持つ動的システムを実現する。
