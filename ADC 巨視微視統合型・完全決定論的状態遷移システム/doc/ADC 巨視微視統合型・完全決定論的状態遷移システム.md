# 巨視微視統合型・完全決定論的状態遷移システム（Axiom Dynamics Core）設計書

## 1. 概要

本設計書は、ナノスケールの微視的挙動から惑星規模の巨視的空間までを同一空間で完全決定論的にシミュレーションするための汎用フレームワーク「Axiom Dynamics Core (ADC)」の仕様を定義する。極端な制約環境下におけるシステム設計思想を基盤とし、64bitアーキテクチャへの移行、微視的な周期動態（CVD）、巨視的なライフサイクル（CVM）、および空間環境の波動伝播（FBM）を統合した「多層的周期構造」により、演算効率と知覚的リアリティを極限まで拡張する。

本フレームワークは以下の特徴を持つ：

* 状態を64bit整数（`BigUint64Array`）のビットパッキング構造として扱う高解像度データモデル
* WGSLのALU負荷を最適化した、浮動小数点誤差皆無の完全GPUバウンドな階層型宇宙規模空間ハッシュグリッド
* 二重周期構造（CVMによるライフサイクル管理とCVDによる心拍動態）による演算負荷の段階的削減
* 内向き位相力学（RCM）を用いた非観測領域の次元圧縮、および絶対Tick数をシードとした完全決定論的再展開
* GPU内ダウンサンプリング（Reduction）とステージングバッファ経由の非同期リードバックによるCPUストール回避
* 外周場力学（FBM）と、Hash-Grid Chunk SDFを用いた動的かつフラットな省メモリ地形化・環境波動伝播
* 知覚力学エンジン（PDE）による、内部ロジックとユーザー体験（UX）の高度な翻訳・統合制御
* 空間エントロピー閾値に基づく動的データ・リアライメントによるL1/L2キャッシュヒット率の維持とVRAM帯域の最適化
* 観測者効果に基づくLOD制御（固定長プールによる動的実体化と確率雲への圧縮）
* ネットワーク合意に基づくシミュレーションTickレートの動的制御（Time Dilation）
* ネットワーク合意された共通シードに基づく、現実世界のエントロピー（環境ノイズ）の決定論的な内因化
* 整数オーバーフローを用いた位相跳躍演算による $O(1)$ での未来予測プロセスとGPUオフロード
* LUT（ルックアップテーブル）と階層構造を維持した高速な64bit固定小数点SoA演算
* 空間分割型P2Pネットワークによるスケーラブルな権限移譲、およびTickベースの入力同期（Input Sync）とロールバック処理による帯域幅の極限最適化
* 動的SDFからの階層Zバッファ（Hi-Z）リアルタイム生成と、Two-Pass Occlusion Cullingを用いたCPU介介在ゼロの描画パイプライン

## 2. システム全体構造

システムは以下の要素で構成され、コアロジック、知覚制御、圧縮・要約、描画制御、未来予測、P2P通信は独立したプロセスで並行実行される。

$$System = (S, A, C, P, R, F, M, V, T, N)$$

* **$S$ (State)**: 64bitビットパックされた個体の内部状態ベクトル。
* **$A$ (Action)**: 状態を変更するための入力。
* **$C$ (Core)**: 状態遷移規則、CVMライフサイクル管理、およびCVD周期積分エンジン（個体動態）。
* **$P$ (Perception)**: PDE知覚変換層。内部状態を五感フィードバックとUIへ翻訳する。
* **$R$ (Regulation/Compression)**: RCMによる状態の次元圧縮、GPU内Reduction演算による要約ベクトルの生成、および絶対Tickに基づく状態復元。
* **$F$ (Field)**: FBMに基づく環境場。Hash-Grid Chunk SDFによる固定長プール管理と軽量化された波動方程式による空間グリッド。
* **$M$ (Memory)**: 64bitパッキングSoA、GPUリングバッファ履歴、および演算用LUT。
* **$V$ (View)**: 描画アダプタ（WebGPU）および演出用エフェクト。
* **$T$ (Time)**: グローバル合意に基づくシミュレーションTickレートの動的制御（Time Dilation）。
* **$N$ (Network)**: 空間ハッシュグリッドに基づく権限分割型 WebRTC P2P プロトコルとTickベース入力同期。

## 3. メモリレイアウトと空間構造

### 3.1 64bitビットパッキング（Bit-packed SoA）

各エンティティの状態は `BigUint64Array` 内にパッキングされる。CVMのライフサイクル状態とCVDの位相を統合し、生存状態に応じた演算スキップを可能にする。

| Bit範囲 | 割当 | 精度 / 用途 |
| :--- | :--- | :--- |
| `0-15` | **Normalized Phase** | CVD位相状態（16bit）。0.0〜1.0を65536段階で表現。 |
| `16-31` | **Intensity** | 強度・熱量（16bit）。周囲への伝搬エネルギー。 |
| `32-47` | **Fatigue** | 疲労蓄積（16bit）。CVMの成長・衰退に直結。 |
| `48-51` | **Lifecycle State** | CVM状態（誕生、成長、充実、変化、完了、おやすみ）。 |
| `52-52` | **Drive/Rest Flag** | CVD活動状態フラグ。演算Early Exitの判定に使用。 |
| `53-63` | **System Flags** | Fixed mode設定、LOD、生存フラグ、遺伝子情報。 |

### 3.2 宇宙規模GPU空間ハッシュ

空間的な近傍個体を特定するためのMorton Code生成において、シェーダーのALU負荷と精度要求のバランスを最適化する。全体のハッシュ幅をWGSLのネイティブ精度やJavaScriptの最大安全整数に収まる51bit幅とするか、あるいは階層的空間グリッド（広域を定義する粗い32bitハッシュと、局所を定義する16bitハッシュのハイブリッド等）を採用する。WebGPU（WGSL）側では `vec2<u32>` を用いたエミュレーションにより、プラットフォーム間での完全なビット一致を保証する。

## 4. 時間・空間・負荷制御

### 4.1 二重カリング機序：Standby と Rest

演算リソースを動的に最適化するため、以下の段階的なカリングを適用する。
* **マクロカリング（CVM層）**: `Standby`（おやすみ）状態にあるエンティティは、GPUバッファから除外され演算を完全に停止する。
* **ミクロカリング（CVD層）**: 活動中であっても `Rest`（休憩中）相にある個体、または変化率が極小の個体は、Compute Shaderの早期退出（Early Exit）により相互作用計算をスキップする。

### 4.2 観測者効果と意味的圧縮・再展開（RCM層）

非観測領域のエンティティ群は、個別の64bitデータとしての演算を停止し、RCMモデルに基づく次元圧縮を適用する。
* **Schrödinger's Culling**: 画面外の個体群を空間ハッシュの階層構造に基づきクラスタリングし、少数の「重心（Core）」へと集約することでVRAM消費と計算コストを最小化する。
* **決定論的状態復元**: 観測者が再びその領域に視点を向けた際（波束の収束時）には、空間ハッシュ座標とネットワークで完全合意された「シミュレーションの絶対Tick数（Global Tick Count）」をシードとした擬似乱数（PRNG）を用い、圧縮された重心データから元の64bit個体群の位相や疲労度を $O(1)$ で決定論的に再展開する。OSのシステム時間は分岐の原因となるため一切参照しない。
* **情報の要約と非同期リードバック**: 圧縮されたデータから「現在の世界のトレンド（熱量が高い座標や主要な変化）」を抽出し、LLMエージェントが理解可能な自然言語コンテキストとして出力する。この際、GPUのCompute Shaderを用いて数十バイト程度の極小の要約ベクトルへ事前にダウンサンプリング（Reduction演算）を行う。このデータを遅延許容型のステージングバッファ（Staging Buffer）経由で数フレーム遅れで非同期リードバック（`mapAsync`）することで、GPUパイプラインのストールを完全に回避する。

### 4.3 位相跳躍による $O(1)$ 予測

未来予測プロセス（Shadow Worker）において、周期特性を利用した位相跳躍を適用する。
$$ShadowPhase = (CurrentPhase + \frac{FutureStep}{Cycle}) \pmod 1$$
実装仕様としては、WGSLにおける16bit空間（0〜65535）の加算に伴う自然な整数オーバーフロー（ビットラップアラウンド）を $\pmod 1$ として利用する。これにより浮動小数点演算を排除しつつ、中間フレームの逐次計算を省略し、任意の未来時点におけるエンティティの状態を $O(1)$ のコストで算出する。

### 4.4 局所時間拡張（Time Dilation）とTick同期

ネットワーク上の完全決定論的ロックステップ（Tick同期）を維持するため、Time Dilationは各ノードのローカル処理による遅延ではなく、ネットワークの合意形成に基づくグローバルなシミュレーションTickレート（$dt$）の動的増減として実装する。これにより、負荷に応じた処理時間の引き伸ばし（スローモーション化）を行いつつ、空間ハッシュの相互作用における決定論を完全に担保する。

## 5. 環境・地形制御（Field Layer）

### 5.1 外周場力学（FBM）とエントロピーの合意形成

空間全体を軽量なグリッドとして定義し、FBMに基づく波動方程式を適用する。デバイスの温度変動やマイク入力、社会的な外部指標などを空間を伝わる「波紋」として内因化する際、各ノードのローカルな入力を直接用いることはしない。P2Pネットワークのホスト（または合意されたノード）が収集した環境ノイズを「共通の決定論的シード」として全ノードに配信し、全ノードが同一の入力値として処理することで、完全決定論の破綻（Desync）を防ぐ。

### 5.2 死骸の動的地形化とHash-Grid Chunk SDF

寿命を迎えた個体は、動的SoAバッファから抹消され、環境場のSDF（符号付き距離関数）へと不可逆的に書き込まれる。無限の空間でのVRAM枯渇を防ぎ、WGSLにおける動的メモリ割り当てや再帰処理の制限を回避するため、SDFのデータ構造には「空間ハッシュに基づく固定長プール型のチャンクSDF（Hash-Grid Chunk SDF）」を採用する。空間を固定サイズのチャンク（例: $16 \times 16 \times 16$ ボクセル）に分割し、死骸が発生したチャンクのみを事前に確保したフラットなVRAMプールから割り当て、空間ハッシュマップでインデックスを引く。これにより、何もない空間のメモリ消費をゼロに抑えつつWGSL上で高速に地形を形成する。さらに、この動的に変動するSDFからCompute Shaderを用いて階層Zバッファ（Hi-Z）を毎フレームリアルタイムに構築し、後段のTwo-Pass Occlusion Cullingの精緻な判定基準として利用する。

## 6. 知覚制御とインターフェース（Perception Layer）

### 6.1 専門用語の完全秘匿と翻訳

内部ロジックの専門用語は、UI上には一切表示しない。PDE（知覚力学エンジン）が以下の変換表に基づき、誰にでも直感的で分かりやすい言葉に変換して提供する。パラメータ調整にはスライダーを用い、操作性を最適化する。

* **CVM状態** → 「誕生、成長、充実、変化、完了、おやすみ」
* **CVD動態** → 「活動中 / 休憩中」
* **Fatigue / Intensity** → 「つかれ / エネルギー」
* **Cycle / Phase** → 「リズム / 進み具合」
* **Field (FBM)** → 「空間のゆらぎ / 周囲の環境」

### 6.2 統合フィードバックとダイアログ

PDEは内部状態を10段階の「活動レベル」に正規化し、視覚・触覚・音響を制御する。UI上に表示されるAIキャラクターのメッセージやシステムログのサンプルは、プレースホルダーを使用せず、実際にLLMを通した自然なテキストを出力して生成する。

### 6.3 固定モード（Fixed mode）

標準の観測モードとして「Fixed mode」を採用する。これは対象座標に対してカメラをロックし、ユーザーの明示的な操作以外による視点の自動リセットや位置移動を完全に無効化する仕様である。

## 7. 実装および決定論の担保

### 7.1 固定小数点演算とLUTを用いた超越関数の最適化

ネットワーク同期の不一致を防ぐため、WGSL内の全ての物理・動態演算（FBM空間波及を含む）から浮動小数点を排除する。`vec2<u32>` を用いた64bit固定小数点（32.32フォーマット等）を標準とし、ハードウェア間の丸め誤差を根絶する。また、波動や円軌道計算に必須となる超越関数（`sin` や `cos` 等）については、WGSLのネイティブ関数（`f32`）を使用せず、決定論的かつ事前に計算された固定小数点のルックアップテーブル（LUT）と軽量な線形補間を組み合わせて自前実装し、ALU負荷の爆発と精度落ちを防ぐ。

### 7.2 動的データ・リアライメント

個体の移動によるMorton Codeの乱れ（空間エントロピー）を監視し、その乱れが一定閾値を超えた場合にのみ、非同期でSoAバッファの物理的な並び替え（ソート）を実行する。空間的に近いエンティティをメモリ上で近接配置しL1/L2キャッシュのヒット率を維持しつつ、巨大なVRAM帯域幅を消費するGPUソート処理の実行頻度を最適化することで、64bit化による帯域負荷を相殺する。

## 8. ネットワーク・スケーラビリティ

### 8.1 空間分割型P2Pアーキテクチャ

純粋なフルメッシュ構造による通信負荷の限界（$O(N^2)$）を回避するため、空間ハッシュグリッドの領域ごとに権限（Authority）を分割したWebRTC通信（空間分割型P2PまたはStar/Tree型トポロジ）を採用する。これにより、ユーザーが密集した際にも各ノードの接続数を適切に制限し、宇宙規模のシミュレーションにおけるネットワークのスケーラビリティを担保する。

### 8.2 Tickベース入力同期（Input Sync）とロールバック

巨大なエンティティ状態（SoAバッファ）を直接同期することによるネットワーク帯域の崩壊とMTU制限の超過を防ぐため、ネットワークで同期するデータは「ユーザーの入力（Action）と環境シード」のみに限定する。完全決定論的システムの特性を活かし、絶対Tickに紐づけた入力同期（Input Sync）および必要に応じた予測・巻き戻し処理（Rollback）を実行する。さらに、各ノードで定期的に状態のハッシュ値を突き合わせることで、極低帯域でDesync（同期ズレ）を検知・補正する。
