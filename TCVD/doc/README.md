# Triadic Velocity Dynamics (TVD)

Triadic Velocity Dynamics（TVD）は、3つの要素（Drive / Regulation / Field）の相関関係によってシステムの動態を制御する数理モデルエンジンです。

## 1. コンセプト：DRFモデル
本モデルは、以下の3要素によるダイナミクスをプロダクトのコアロジックとして提供します。

* **Drive (D)**: 外向きの推進力、活性度
* **Regulation (R)**: 内向きの制御、抑制力
* **Field (F)**: 内部状態のコンテキスト（文脈）

これらが相互に干渉し合うことで、単純な線形処理では得られない、生命力のある動的な挙動（NPCの意思決定、アルゴリズムの可視化など）を実現します。

## 2. 状態更新モデル
システムの主要な状態 $X = [D, R, F]$ は、以下の更新式に基づいて時間発展します。

$$X_{t+1} = X_{t} + \Delta t \cdot [\text{Interaction}(D, R, F) - \text{Decay}(X_t)] + \text{Input}$$

### 特徴
* **非線形入力処理**: すべての外部入力は `tanh` 等の非線形関数を通り、システムの安定性を損なわない形で反映されます。
* **環境統合**: 内部状態（Field_internal）と外部環境（Field_external）を厳密に分離し、安全に統合します。
* **数値計算の安定性**: デフォルトで2次ルンゲ＝クッタ法（RK2）を推奨し、高精度なシミュレーションが可能です。

## 3. クイックスタート

### 基本的なパラメータ設定
```python
params = {
    "dynamics": {
        "stability": 0.5,   # 構造的安定性
        "decay": 0.01,       # 動的減衰
        "gain_k": 1.5        # ゲイン感度
    },
    "input_weight": 0.1      # 入力影響度
}
```

### 観測インジケータ
システムの現在の「勢い」や「負荷」を以下の指標でリアルタイムに観測可能です。
* **Drive - Regulation**: システムの純粋な推進力
* **Field**: 現在の環境プレッシャー
* **Activity**: 全体の活性エネルギー（D + R + F）

## 4. 実装のポイント
1.  **スケーラビリティ**: 各インスタンスは独立しており、バッチ処理や並列計算が容易です。
2.  **可観測性**: 内部状態と制御指標が分離されており、デバッグやログ解析がスムーズです。
3.  **ドリフト防止**: 状態空間は $[0, 1]$ にクランプされ、数値的な破綻を防ぎます。
